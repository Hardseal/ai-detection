# 3.13.1: Monitor, control, and protect communications at boundaries

This control is implemented through a combination of policy, procedure,
and technical configuration. The organization has documented the
implementation in the System Security Plan. Evidence of implementation
is tracked in our POA&M and reviewed annually by the responsible role.
Controls are reviewed quarterly and all findings are remediated according
to severity in accordance with NIST SP 800-171 and CMMC Level 2
assessment objectives. Shared responsibility with our external service
provider is documented in the customer responsibility matrix.

## Implementation specifics: Widget Defense Corp

The external boundary for the CUI enclave is enforced by a Palo Alto
Networks PA-3220 HA pair running PAN-OS 11.1.4-h3 at site WD-HQ-01.
Firewall rules are managed as Terraform code in Git repo
`infra-fw-prod` under directory `modules/cui_boundary/`; every change
requires a peer-reviewed pull request and an explicit approval from the
Network Engineer (Martin Obi, martin.obi@widgetdefense.com) before the
GitHub Action `tf-apply-fw-prod.yml` runs.

East-west segmentation inside the CUI enclave is enforced by VLAN 231
with ACLs maintained on the Cisco Catalyst 9500 core stack, configuration
archive `cfg-core-stack-2026-04-07.cfg`. All egress traffic is proxied
through a Zscaler ZIA tenant with TLS 1.3 inspection; Zscaler NSS feeds
flow logs into Splunk index `net_flow` with SHA-256 integrity on each
forwarder (forwarder IDs: fwd01, fwd02, fwd03).

Evidence: tf-apply-fw-prod-run-8812.log, pan-config-2026-04-10.xml,
zscaler-nss-export-2026-04.csv.
