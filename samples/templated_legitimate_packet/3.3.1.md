# 3.3.1: Create and retain system audit logs and records

This control is implemented through a combination of policy, procedure,
and technical configuration. The organization has documented the
implementation in the System Security Plan. Evidence of implementation
is tracked in our POA&M and reviewed annually by the responsible role.
Controls are reviewed quarterly and all findings are remediated according
to severity in accordance with NIST SP 800-171 and CMMC Level 2
assessment objectives. Shared responsibility with our external service
provider is documented in the customer responsibility matrix.

## Implementation specifics: Widget Defense Corp

Splunk Enterprise 9.3.2 on-prem indexes syslog, Windows Event Log, and
AWS CloudTrail events from every in-scope asset in the CUI boundary.
Indexer cluster `cui-idx-cluster-01` (three nodes, each 32 vCPU / 256 GB)
holds 365 days of hot/warm retention; cold-tier archives roll to an
AES-256 encrypted S3 bucket `s3://widgetdefense-cui-audit-cold` with
7-year object-lock retention configured via Terraform module
`modules/audit_archive/`.

Audit events required by 3.3.1[a] (user logons, privilege changes,
policy changes, firewall denies, AV detections, file-integrity changes)
are captured by saved-search `svc_audit_event_coverage` which runs
daily at 06:00 UTC and alerts the SOC Lead (Priya Mehta,
priya.mehta@widgetdefense.com) on coverage drops > 2%. The most recent
coverage report (2026-04-19) showed 99.7% asset coverage with two gaps
(dev-jumphost-03, fw-test-02) tracked as POA&M items POAM-2026-047
and POAM-2026-048, both scheduled for remediation by 2026-05-01.
